# Contributing to Stash

Thanks for your interest in contributing.

## Quick start

1. Fork the repository.
2. Create a branch from `main`.
3. Open `Stash.xcodeproj` in Xcode 26.6+.
4. Run the app and verify your change locally.
5. Open a pull request with a clear summary and screenshots/GIFs for UI work.

## Development notes

- Keep changes focused and small when possible.
- Follow existing naming and file organization under `Stash/`.
- Avoid adding dependencies unless there is a strong reason.
- For UI changes, preserve native macOS behavior and keyboard flows.

## Pull request checklist

- [ ] Build succeeds locally.
- [ ] Behavior is validated manually.
- [ ] README/DOCS updated if behavior changed.
- [ ] No secrets or machine-specific files are included.

## Reporting issues

Please include:

- macOS version
- Xcode version
- Reproduction steps
- Expected behavior
- Actual behavior
- Screenshots/logs when relevant
